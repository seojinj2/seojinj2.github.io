# OSI 7계층 정리 및 보안 관점 분석

## 1. 개요

**OSI(Open Systems Interconnection) 7계층 모델**은 국제 표준화 기구(ISO)가 정의한 네트워크 통신 표준 구조로, 데이터가 한 시스템에서 다른 시스템으로 전송되는 과정을 7단계로 구분한다.
이 모델은 각 계층이 독립적인 기능을 수행하도록 설계되어 **네트워크의 안정성, 신뢰성, 유지보수성**을 높인다.

### 🔹 OSI 모델의 주요 장점

* **모듈화(Modularity)**: 한 계층 변경이 다른 계층에 영향을 주지 않음
* **표준화(Standardization)**: 다양한 시스템 간 호환성 보장
* **간소화(Simplification)**: 복잡한 네트워크 통신 구조를 단계별로 단순화

---

## ⚙️ 2. OSI 7계층별 역할 및 보안 관점

### ① 물리 계층 (Physical Layer)

> **전기적·기계적 데이터 전송 담당**

| 구분      | 내용                     |
| ------- | ---------------------- |
| 전송 단위   | 비트(Bit)                |
| 주요 장비   | 케이블, 허브, 리피터, 광섬유      |
| 주요 프로토콜 | RS-232, Ethernet, FDDI |

**보안 관점**

* 공격: 케이블 도청, 무선 신호 가로채기
* 방어: 물리적 접근 통제, 신호 암호화, 전자기 차폐

---

### ② 데이터링크 계층 (Data Link Layer)

> **인접 노드 간 데이터 전송 및 오류 검출·수정**

| 구분      | 내용             |
| ------- | -------------- |
| 전송 단위   | 프레임(Frame)     |
| 주요 장비   | 스위치, 브릿지       |
| 주요 프로토콜 | Ethernet, HDLC |

**보안 관점**

* 공격: ARP 스푸핑, MAC 플러딩
* 방어: 포트 보안, DHCP 스누핑, 802.1X 인증

---

### ③ 네트워크 계층 (Network Layer)

> **데이터의 라우팅과 경로 선택(IP 기반)**

| 구분      | 내용            |
| ------- | ------------- |
| 전송 단위   | 패킷(Packet)    |
| 주요 프로토콜 | IP, ICMP, ARP |
| 주요 장비   | 라우터           |

**보안 관점**

* 공격: IP 스푸핑, ICMP 플러딩
* 방어: 인그레스 필터링, RPF 검사, 트래픽 모니터링

---

### ④ 전송 계층 (Transport Layer)

> **프로세스 간 신뢰성 있는 데이터 전송 담당**

| 구분      | 내용            |
| ------- | ------------- |
| 전송 단위   | 세그먼트(Segment) |
| 주요 프로토콜 | TCP, UDP      |

**보안 관점**

* 공격: SYN 플러딩, TCP 시퀀스 예측
* 방어: SYN 쿠키, 연결 제한, 시퀀스 무작위화

---

### ⑤ 세션 계층 (Session Layer)

> **통신 세션의 설정·유지·종료 관리**

| 구분      | 내용                |
| ------- | ----------------- |
| 주요 프로토콜 | NetBIOS, RPC, NFS |

**보안 관점**

* 공격: 세션 하이재킹
* 방어: HTTPS 사용, 세션 타임아웃, MFA 적용

---

### ⑥ 표현 계층 (Presentation Layer)

> **데이터의 형식 변환, 암호화·복호화, 압축 담당**

| 구분      | 내용           |
| ------- | ------------ |
| 주요 프로토콜 | SSL, TLS     |
| 주요 기능   | 인코딩, 암호화, 압축 |

**보안 관점**

* 공격: SSL/TLS 취약점, CRIME 공격
* 방어: 최신 암호화 프로토콜 적용, 안전한 압축 알고리즘 사용

---

### ⑦ 응용 계층 (Application Layer)

> 사용자와 직접 상호작용하는 서비스 계층

| 구분      | 내용                        |
| ------- | ------------------------- |
| 주요 프로토콜 | HTTP, FTP, SMTP, DNS, SSH |
| 주요 역할   | 사용자 인터페이스, 데이터 송수신        |

**보안 관점**

* 공격: SQL Injection, XSS 등
* 방어: 입력값 검증, WAF 구축, HTTPS 사용

---

## 3. 최신 보안 트렌드 — Cross-Layer Security

최근에는 한 계층의 취약점을 이용해 다른 계층을 공격하는 계층 간 보안 위협(Cross-Layer Attack)이 증가하고 있다.
예를 들어, 응용 계층의 취약점을 통해 네트워크 계층의 라우팅 정보를 조작하는 공격 등이 이에 해당한다.
따라서 계층 간 통합 보안 접근법(Cross-Layer Security)이 새로운 보안 연구 방향으로 주목받고 있다.

---

## 4. 결론

* OSI 7계층은 네트워크 이해의 기본이자 보안 분석의 핵심 프레임워크다.
* 실제로는 TCP/IP 4계층 모델이 사용되지만, OSI 모델은 개념적 기반으로 여전히 중요하다.
* 각 계층의 역할과 취약점을 이해하는 것은 보안 전문가로 성장하기 위한 필수 단계이다.
